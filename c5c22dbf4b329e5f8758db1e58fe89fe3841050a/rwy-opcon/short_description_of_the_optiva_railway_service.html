Short Description of the Service

Introduction

This section provides an overview of the .

enables railway companies to set up an effective and flexible system for voice and data communication between railway operational staff, such as drivers, dispatchers, shunting team members, train engineers, and station controllers.

The service is based on two pillars: The EIRENE (European Integrated Railway Radio Enhanced Network) numbering plan, and the flexible routing and call distribution options provided by an Intelligent Network (IN).

EIRENE numbering plan

The EIRENE numbering plan complies with the requirements of a railway operation and is based on identification numbers that determine the lines for both trains and persons. For detailed information about the EIRENE numbering plan, see chapter 8 EIRENE Numbering Plan.

Service functionality

The service links the Functional Numbers (FN) defined by EIRENE with the Mobile Subscriber ISDN (MSISDN)of the Global System for Mobile Communications - Railway (GSM-R) telephony network. This makes it possible to route calls in different ways according to the numbering plan. Using Unstructured Supplementary Service Data (USSD) access codes, users can assign FNs to their MSISDNs depending on the function they are working in.

Architecture

Physically, consists of several components or network elements, some of them mandatory, some optional. Logically, consists of several layers of software. The topmost of these layers contains the application delivering the actual functionality, namely controlling the base network. For more information about the architecture, see Embedding in the System.

Access variants

supports dialled access (Mobile Originating Call, MOC), SMS Mobile Originated (SMS-MO), USSD (for registration/deregistration to Functional Numbers etc.), Data Mobile Originated (DATA-MO) and External Subsystem Access (ESX). For information about the access variants, see sections Access to the Service and Access to the Service.

Geographical redundancy

Geographical redundancy means that two IN platforms at two different geographical locations are installed. Both of the IN platforms contain a service to make sure that in the case of a disaster (e.g., an earthquake), the service is still available. Geographical redundancy is an optional feature which can be implemented project-specifically. For more information, see Geographical Redundancy (GeoRed).
