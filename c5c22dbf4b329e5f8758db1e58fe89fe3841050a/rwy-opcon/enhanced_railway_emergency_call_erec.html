enhanced Railway Emergency Call (eREC)

Description

Enahnced Railway Emergency Call feature of service takes part in registration, deregistration and interrogation of eREC subscribers, and is responsible for updating mobile subscribers with current sectors. eREC is in an improvement of core GSM network feature REC, and doesn’t take part in call processing as it is handled by the core network.

Field definitions

The eREC registration string contains additional information:

Table 1. eREC Field definitions
eREC Field_ID eREC Field name Mandatory/Optional
7 LAC M
8 CellID M
9 Lat O
10 Long O
11 Height O
12 Speed O
13 Heading O
14 ET O
15 Distance O

Example 1:

**214*03120055501***EREC0001,2BA3,,,,,,,# is an eREC Service Registration on train number 555 in Netherlands requiring USSD Sector Identity updates. Location information and odometry information is not present.

Example 2:

**214*0336000005001***EREC0001,2BA3,,,,,,,# in an eREC Service Registration on dummy shunting number 000005001 in France requiring USSD Sector Identity updates. Location information and odometry information is not present.

eREC Registration

eREC registration is similar to the standard USSD subscriber registration. Additionaly dummy FN numbers can be used for registration in following cases:

  • For use in shunting when functional number registration is not used.
  • After border crossing when eREC service in the previous country (network) has to be deactivated but the train functional number has to remain active.

The format of shunting functional numbers is IC + CT + LN + FC. For a dummy functional number LN = 00000 and FC = 5001.

Example: 0336000005001 (IC = 033, CT = 6, LN = 00000, FC = 5001).

The format of train functional numbers is IC + CT + TN + FC. For a dummy functional

number LN = 00000 and FC = 01.

Example: 03360000001 (IC = 033, CT = 2, TN = 00000, FC = 01)

eREC Deregistration

eREC deregistration does not contain any eREC parameters.

Example 1:

##214*03220055501***BULK02 0203 EREC# is an eREC Service Deregistration in Belgium on train number 555

Example 2:

##214*04920000001***EREC# is an eREC Service Deregistration in Germany at border crossing. Since it is not intended to deregister the train number, the dummy train number 0000001 is used for eREC Service Deregistration.

eREC SectorID Update request

eREC SectorID Update request is done by sending the same USSD string as for eREC Registration. For details please refer to eREC Registration section.

Note: Every registration string sent by an already registered subscriber will be treated as an SectorID Update request, and not a re-registration message. To perform a re-registration, subscriber has to deregister and register again using appropriate USSD strings.

eREC SectorID Update indication

eREC SectorID Update indication can be triggered by subscriber (via SectorID Update Request) or by service (after Train Location Update). The structure of the message is: ##214*EREC<+eREC info>#

Example:

##214*EREC126000000,HXUX,204,021,S,21600,86400# is an eREC Sector Identity Update Indication for a train in the Netherlands. Sector 1 is used for eREC call initiation. eREC Sector Identities for call reception are 1, 2 and 6.
