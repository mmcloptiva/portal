Functional Description

Introduction

Call monitoring based FNs

Which calls are recorded is decided in the GSM-R network at the MSC which contains on a database of users which are to be recorded. The database is administrated using a Voice Recording Terminal (VRT).

optionally supports voice recording based on FNs. To implement this feature, the VRT is connected to the system over an online interface (TCP/IP). The system informs VRT over this interface about FN registrations (FN-MSISDN mappings) and deregistrations by sending an FNMobileRegistrationUpdate request message to the VRT. This request message contains the FN, the MSISDN, and expiry information. In the case of deregistrations, the MSISDN is 0 and the expiration date is 0, which informs the VRT that the formerly registered MSISDN has been deregistered.

Note, that the VRT can ignore the expiration date information as the system will send an FNMobileRegistrationUpdate request message at the deregistration anyway.

The VRT returns an FNMobileRegistrationUpdate response for each request. The response message contains information indicating whether the information about the registration/deregistration was successfully processed.
