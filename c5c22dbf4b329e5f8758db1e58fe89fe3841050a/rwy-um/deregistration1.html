Deregistration

Introduction

This section provides a detailed description of deregistration from FNs via USSD and shows the corresponding service data. It also provides a short description of how a deregistration is performed manually via the Web-based GUI.

For a description of the feature, see Deregistration.

Deregistration

Three types of deregistration can be distinguished:

  • Deregistration by the users themselves
  • Deregistration of users by the system after timeout, i.e. after the expiration date of the MSISDN - FN assignment has been exceeded
  • Forced deregistration

The following access string has to be entered for deregistering from a FN (<> indicates placeholders):

##<SC>*<FN>***#

Table 1. Deregistration by user
Placeholder Description
SC Service code; defined by EIRENE to be 214
FN Functional Number (FN) in international EIRENE format (IC/RAC + FN)

The initial part of this string (##<SC>) is defined in the Ussd Deregistration String field.

The following example shows the access string for a user who deregisters from the FN

2 12345678 01

(with IC/RAC = 047):

##214*04721234567801***#

Bulk deregistration

Only the cab radio and the lead driver (FC=01) are allowed to perform bulk deregistrations.

If bulk operations are to be used by the cab radio, it must be configured to allow network override.

The following access string has to be entered for bulk deregistration of FNs:

<OC><SC>*<LEAD_FN>***<BULK_TAG><NUM_FC><FC_LIST>#

Table 2. Bulk deregistration of FNs
Placeholder Description
OC Operation code, ## for deregistration
SC Service code; defined by EIRENE to be 214
* Field delimiter character
LEAD_FN Functional number of the lead driver (FC must be 01)
BULK_TAG Text tag which indicates a bulk operation: BULK
NUM_FC Digit which specifies the number of FCs which follow
FC_LIST List of two-digit FCs
# USSD string terminator

The following example shows the access string for the deregistration of four FNs (FC=01, 02, 03 and 04) when the lead FN is 03121234501:

##214*03121234501***BULK3 020304#

The response message sent by the system has the following format:

<OC> <NUM_SFC> <FC_SLIST> <NUM_UFC> <FC_ULIST>

Table 3. Bulk deregistration response message
Placeholder Description
OC Operation code (outcome) 12 for bulk deregistration
NUM_SFC Number of successfully deregistered FCs
FC_SLIST List of successfully deregistered FCs
NUM_UFC Number of unsuccessfully deregistered FCs
FC_ULIST List of unsuccessfully deregistered FCs

Deregistration after timeout

In the case of deregistration after timeout, the deregistered user is informed that he has been deregistered via the following string:

##<SC>*<FN>*88*<HelpDeskMSISDN>*<addInfField>#

Table 4. Deregistration after timeout
Placeholder Description
SC Service code; defined by EIRENE to be 214
FN Functional Number (FN) from which the user has been deregistered
HelpDesk MSISDN MSISDN of the help desk, specified in the Ussd Help Desk Msisdn field.
addInfField Additional information, specified in the Ussd Additional Info Field field.

Forced deregistration

The following access string has to be entered in the case of forced deregistration:

##<SC>*<FN>*<SupervisorIndicator>*<MSISDN>*#

Table 5. Forced deregistration
Placeholder Description
SC Service code; defined by EIRENE to be 214
FN FN in international EIRENE format (IC/RAC + FN)
Supervisor Indicator Supervisor indicator, specified in the Ussd Supervisor Indicator field.
MSISDN MSISDN currently registered to the FN

The following example shows the access string for a user who is deregistered from the FN 21234567801 by another user with the MSISDN 0047855555555 (with IC/RAC = 047):

##214*04721234567801*88*0047812345678*#

The deregistered user is informed that he has been deregistered via the following string:

##<SC>*<FN>*<SupervisorIndicator>*<FDMSISDN>*<addInfField>#

Table 6. Deregistered user string
Placeholder Description
SC Service code; defined by EIRENE to be 214
FN FN from which the user has been deregistered
Supervisor Indicator Supervisor indicator, specified in the Ussd Supervisor Indicator field.
FDMSISDN MSISDN of the user who initiated the forced deregistration
addInfField Additional information, specified in the Ussd Additional Info Field field.

The deregistered user will be informed via the following USSD string:

##214*04721234567801*88*0047855555555*addInfField#
