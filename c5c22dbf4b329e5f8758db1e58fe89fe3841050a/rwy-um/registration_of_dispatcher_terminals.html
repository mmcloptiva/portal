Registration of Dispatcher Terminals

Description

An employee can register the ISDNNo of a dispatcher terminal to an FN. The registration is done using the dispatcher terminal. To start the registration, the CS sends a request message to the SEP via the MGW to perform the registration. The request message includes the following information:

  • the ISDNNo of the dispatcher terminal
  • the FN that is supposed to be assigned to the ISDNNo

The SEP checks the parameters contained in the message and the standard registration is performed. If the registration was successful, the SEP sends a message to the CS to acknowledge the registration. Otherwise, the SEP sends a message to the CS which indicates that the registration was not successful.

Registration

The following messages are sent between the CS and the SEP:

  • Request message to register the ISDNNo of the dispatcher terminal to the FN (CS -> SEP)
  • Response message to acknowledge the registration (CS <- SEP), or
  • Reject message with reject reason (CS <- SEP)
