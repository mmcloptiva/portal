Functional Addressing

Description

Railway operation requires communication between users based on their current roles rather than communication based on the numbers assigned to the communication equipment they are using. This is achieved by the so-called functional addressing, providing a mechanism by which a communication terminal (e.g. mobile phone) can be addressed by a number identifying the function for which it is currently used.

Functional Numbers

These Functional Numbers (FN) are defined in a numbering plan defined by EIRENE as a standard for the GSM-R networks of the European railway companies.

In the EIRENE numbering plan, FNs have the following structure:

The different segments have the following meaning:

  • The Call Type (CT) prefix is used to distinguish between the different types of user numbers that are allowed within the national EIRENE numbering plan. It is an indication to the network of how to process the number dialled.
  • The User Identifier Number (UIN) is the main identifier of, e.g., a train, a shunting team, etc.
  • The Function Code (FC) is used as an identification of, e.g., the person or equipment on a particular train, or a particular team within a given area.

EIRENE Numbering Plan describes the exact structure of an FN together with some examples of these numbers.

A user, either from within the GSM-R or from the fixed network, dials an FN to reach the conductor of a certain train. The MSC triggers the service which searches the database for the MSISDN to which the FN is currently assigned and returns this MSISDN to the MSC. The MSC then routes the call appropriately.

Calling number presentation

If the calling party is a GSM-R user, the system can be configured to display the caller’s FN (rather than his MSISDN) on the communication device of the called party (Calling Line Identification Presentation (CLIP)).

Different length of the FNs

supports the use of FNs of different length. This applies for FNs of different length used within one network and for the case that a train moves between countries using FNs of different length.

Call monitoring based on FNs

Call monitoring based on FNs is an optional feature and enables voice recording based on an FN. For example, the voice communication between dispatcher and train personnel can be recorded for later evaluation (accident analysis).
