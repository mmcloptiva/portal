Functional Description

Introduction

This section provides a functional description of the interrogation mechanisms via USSD.

Interrogation

The following table provides an overview of how USSD interrogation is handled:

Stage Description
1 If a user interrogates an FN, i.e. a user wants to find out which MSISDN is registered to a specific FN, the service searches the database for the FN, and, if it finds an MSISDN, sends it to the interrogating user.
2 If the user himself performs the interrogation (MSISDN interrogation) he enters the Ussd Interrogation2 String. The system checks whether the MSISDN belongs to a cab radio (Cabradio Flag is switched on), in which case the interrogation is aborted.
3 If the MSISDN does not belong to a cab radio, the system iterates over the FNs in the Function Register dataset that have user’s MSISDN in MSISDN field.
4 A USSD string containing the FN(s) the user is assigned to is sent to the user.
